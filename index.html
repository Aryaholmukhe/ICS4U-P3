Algorithm Analysis

Algorithm Best Case Worst Case Why Best Case is Fast Why Worst Case is Slow
Bubble Sort O(n) O(n²) The list is already sorted, requiring only one pass with no swaps. The list is sorted in reverse order, requiring the maximum number of comparisons and swaps.
Quick Sort O(n log n) O(n²) The pivot splits the list into two equal halves at each step, minimizing recursion depth. The pivot is always the smallest or largest element, leading to the most unbalanced partitions and maximum recursion depth.
Binary Search O(1) O(log n) The target is found on the first comparison (middle of the sorted list). The search repeatedly halves the list until the target is found or the list is exhausted, taking log(n) steps.
Linear Search O(1) O(n) The target is the first element in the list, requiring just one comparison. The search must check every element, requiring n comparisons if the target is the last element or not present.

Binary Search: Efficient for sorted lists; halves the search space with each step. Best case: target is at the middle. Worst case: logarithmic steps are needed.
Linear Search: Works on unsorted lists. Best case: target is the first element. Worst case: must traverse the entire list.
